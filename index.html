FAQs

What is React.js and explain the concept of "components" in React.

React.js is a JavaScript library for building user interfaces, especially for single-page applications. It enables developers to create reusable UI components. Components in React are independent, reusable pieces of UI that can be combined to build complex user interfaces. A component can be either a function or a class and returns React elements describing what should appear on the screen.

What is JSX in React, and how does it work?

JSX (JavaScript XML) is a syntax extension for JavaScript used in React to describe the UI. JSX allows you to write HTML-like code inside JavaScript files, which is then transformed into regular JavaScript function calls by Babel (a transpiler). JSX makes it easier to visualize the structure of the UI and integrates seamlessly with JavaScript logic.

What is the Virtual DOM, and how does React use it to optimize performance?

The Virtual DOM is an in-memory representation of the real DOM elements generated by React components. When the state of an object changes, React first updates the Virtual DOM. Then, it compares the Virtual DOM with the previous version using a process called "reconciliation" and efficiently updates the real DOM, applying only the changes. This optimization reduces the need for costly DOM manipulations and enhances performance.

Explain the concept of "props" in React and how they are used.

Props (short for "properties") are inputs passed to React components from their parent components. Props are used to pass data and event handlers to child components, allowing for reusable components with customizable data. Props are immutable, meaning they cannot be modified by the receiving component and are used to render dynamic data.

What is "state" in React, and how does it differ from props?

State is a special object in React components that stores data that can change over time and affect what is rendered on the screen. Unlike props, which are passed to the component and are immutable, state is managed within the component itself and can be updated by the component. When the state changes, the component re-renders to reflect the updated data.

Explain the useState hook and provide an example of how it is used.

The useState hook is a React hook that allows functional components to have state. It returns an array with two elements: the current state value and a function to update it. Here is an example:

const [count, setCount] = useState(0);

function increment() {
setCount(count + 1);
}

return (
<div>
  <p>Count: {count}</p>
  <button onClick={increment}>Increment</button>
</div>
);
What is the purpose of the useEffect hook in React, and when would you use it?

The useEffect hook is used to perform side effects in functional components, such as fetching data, subscribing to events, or manipulating the DOM. It takes a function as an argument and runs after the component renders. You can also pass dependencies to useEffect so it runs only when specific values change. Example use cases include data fetching or setting up timers.
